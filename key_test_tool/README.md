# Windows按键测试工具

用于测试不同Windows按键模拟方法的有效性，帮助排查自动按键在特定应用上无法工作的问题。

## 支持的按键方法

1. **PyAutoGUI** - 你项目中正在使用的方法
2. **pynput** - 第三方按键库
3. **keybd_event** - Windows API (较旧但兼容性好)
4. **SendInput** - Windows API (推荐方法)
5. **PostMessage** - Windows消息队列方式
6. **SendMessage** - Windows同步消息方式

## 使用方法

### 方法一：直接运行Python脚本
```bash
pip install pyautogui pynput pywin32
python key_test_tool.pyw
```

### 方法二：打包为exe后运行
1. 双击运行 `build.bat`
2. 等待打包完成
3. 在 `dist\KeyTestTool` 文件夹中找到 `KeyTestTool.exe`

## 功能说明

- **测试按键**: 支持 a-z, 0-9, enter, space, tab 等
- **按键延迟**: 10-1000ms 可调（默认300ms）
- **测试次数**: 1-20次 可调（默认5次）
- **F10快捷键**: 全局快捷键触发测试
- **日志记录**: 实时显示测试结果到界面，同时保存到 key_test.log 文件

## 测试步骤

1. 运行工具
2. 选择要测试的按键（如 a, enter, space 等）
3. 勾选要测试的方法（默认全部未勾选）
4. 点击"执行测试"按钮或按 F10 快捷键
5. **快速切换到目标应用窗口**
6. 观察目标应用是否有按键反应
7. 在结果窗口中查看哪些方法显示"✓有效"

## 界面按钮说明

- **执行测试 /F10**: 开始执行测试
- **清空结果**: 清空日志显示
- **全选**: 勾选所有方法
- **取消全选**: 取消勾选所有方法

## 结果解读

- **✓有效**: 该方法可以成功向目标应用发送按键
- **✗无效**: 该方法无法影响目标应用（可能被应用屏蔽或权限不足）

## 日志文件

测试过程中会生成 `key_test.log` 文件，记录所有测试操作，便于排查问题。

## 常见问题

### Q: 所有方法都无效
A: 请尝试：
1. 以管理员身份运行测试工具
2. 确认目标应用不是以管理员身份运行（或者也以管理员身份运行工具）

### Q: PyAutoGUI有效但其他无效
A: 这是正常情况，说明你的环境支持PyAutoGUI

### Q: 某些方法报错
A: 某些方法可能需要特定的权限或环境支持

### Q: F10快捷键不生效
A: 确保程序有焦点，pynput库正常工作

## 权限要求

- 建议以管理员身份运行
- 如果目标应用以管理员运行，测试工具也需要管理员权限
